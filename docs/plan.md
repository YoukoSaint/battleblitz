## 3. 阶段划分

### 3.1 总览（5 阶段 + 1 横切）

| 阶段 | 名义周次 | 主要内容 | 风险等级 |
|------|----------|----------|---------|
| **P0 修整** | W1 | 死代码 / 剧情扩容 / 单测补 | 🟢 低 |
| **P1 核心** | W2-W3 | 敌我分阶段 P0.1 + 地形 P0.4 | 🟡 中（架构级改动）|
| **P2 深度** | W4-W5 | 武器耐久 P1.1 + 兵种克制 P1.2 | 🟡 中（数值平衡）|
| **P3 高级** | W6-W8 | 胜利条件 P2.3 + 迷雾 P2.1 + CO Power P2.2 | 🟠 中高（功能多）|
| **P4 收尾** | W9+ | 支援/招募/动画/动态地形 P3.x | 🟢 低（独立）|
| **横切 RT** | 任意 | WS 网关 + 性能 + 测试加固 | 🟡 中 |

### 3.2 详细阶段计划

#### P0 修整期（5-7 天）

| 任务 | 工作量 | 产出 | 验收 |
|------|--------|------|------|
| 死代码清理（`dead-code.md`）| 0.5 天 | -16 imports, -8 aliases | pytest 全过、grep 校验 |

#### P1 核心回合制（5-7 天）

**P1.1 敌我分阶段（2-3 天，重点）**

依赖：无 | 阻塞：P0.4 / P2.x / P2.2 / 横切 WS 网关

| 子任务 | 工作量 | 文件 |
|--------|--------|------|
| Game.phase 字段迁移 | 0.5 天 | models.py |
| 重构 end_turn → phase 切换 | 0.5 天 | routes/turns.py |
| AI 阶段逐帧播放（sleep 0.8-1.2s）| 0.5 天 | routes/turns.py + game_logic.py |
| 阶段判定校验（enemy_phase 拒绝人类操作）| 0.5 天 | routes/actions.py |
| 前端：阶段 banner + 操作灰化 | 0.5-1 天 | app.js + style.css |
| 集成测试 | 0.5 天 | tests/ |

**验收**：4 人局，玩家回合结束 → 自动进入敌方阶段 → AI 逐帧行动 → 切换下一玩家回合

**P1.2 地形增强（3-4 天）**

依赖：P1.1（地形结算在阶段切换时触发）

| 子任务 | 工作量 | 文件 |
|--------|--------|------|
| 新增 6 种地形 + config | 0.5 天 | config.py |
| 城市/工厂/王座特殊效果 | 1 天 | game_logic.py |
| 6 个预设地图重画 | 1 天 | routes/game.py (map presets) |
| 前端：地形色 + 道路连通路标 | 0.5-1 天 | style.css + app.js |

**验收**：mountain_pass 出现"道路"格子，1 MP 可走 2 格；站在城市上每回合回 10% HP

## 6. 风险与缓解

| 风险 | 等级 | 缓解策略 |
|------|------|---------|
| **P1.1 敌我分阶段破坏现有混合轮流** | 🟠 高 | 先写完整集成测试覆盖原行为；切换期保留旧逻辑为 fallback flag |
| **P2.x 数值平衡需要多轮调优** | 🟡 中 | 留 0.5-1 天 buffer；先发内部测试服 |
| **主线剧情扩容依赖剧情设计** | 🟡 中 | 复用 chapter_01 模板，开头/战斗/收尾 3 段式 |
| **WS 网关替换轮询引发前端 bug** | 🟡 中 | 双轨运行（轮询+WS），灰度切换 |
| **P3 高级功能堆积推迟 M2** | 🟡 中 | 严格按价值密度排：P2.3 胜利条件优先 |
| **Llm API 配额/费用** | 🟢 低 | 规则 AI 兜底已就绪，可随时关 LLM |
